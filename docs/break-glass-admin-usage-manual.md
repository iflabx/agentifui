# Break-Glass 管理员使用手册

## 1. 强制红线（最高优先级）

- `break-glass` 管理员账号必须使用 **IdP 不会出现的专用邮箱**。
- 禁止与企业统一身份目录（OIDC/OAuth/CAS）中的任何用户邮箱重复。
- 若该邮箱已被 break-glass 本地账号占用，IdP 同邮箱登录会被系统拒绝（`account_not_linked`），并要求改用密码登录。

## 2. 目的

- 避免本地密码账号与 IdP 账号发生隐式合并/绑定。
- 确保在 IdP 故障、网络隔离或第三方认证不可用时，仍可通过本地方式进入系统执行应急运维。

## 3. 落地要求

- 专用邮箱建议使用内部保留域名，例如：`breakglass-admin@local.internal`。
- 该邮箱不得用于日常办公邮箱系统，不得被任何 IdP 同步策略纳入。
- 若发现邮箱冲突风险，必须立即更换 break-glass 邮箱并重置凭据。

## 4. 验收检查

- 在 IdP 用户目录中检索，确认不存在同邮箱账号。
- 在系统用户数据中检索，确认该邮箱仅对应单一 break-glass 管理员。
- 反向验证：使用 IdP 对同邮箱发起登录，预期被拒绝并回到登录页提示改用密码登录。
- 演练：在禁用 IdP 登录条件下，验证 break-glass 账号可独立登录。
